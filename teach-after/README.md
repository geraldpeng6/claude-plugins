# teach-after

事后复盘教学插件 - 在功能实现或 bug 修复后进行系统性复盘与深入理解。

## 适用场景

当你完成了一个任务（修复 bug 或实现功能）后，想要：
- 深入理解整个过程的前因后果
- 总结可迁移的经验教训
- 避免下次踩同样的坑

## 安装

将 `teach-after` 目录复制到你的 Claude Code 插件目录，或使用：

```bash
claude --plugin-dir /path/to/teach-after
```

## 使用方法

```bash
# 指定复盘主题
/teach-after "XXX bug"
/teach-after "YYY 功能实现"

# 不指定主题，Claude 自动分析上下文
/teach-after
```

## 复盘流程

执行 `/teach-after` 后，Claude 会按以下 6 个步骤进行系统性复盘：

1. **重建时间线** - 梳理目标、失败表现、尝试过程、最终方案
2. **分析失败原因** - 根本原因、问题类型、早期信号
3. **解释成功原因** - 关键改动、因果关系对比
4. **深入讲解原理** - 核心概念、数据流/控制流、代码解析
5. **总结可迁移经验** - 抽象问题模式、生成 checklist
6. **优化建议（可选）** - 当前够用 vs 未来隐患

## 核心原则

- **以理解为目标** - 不是继续改代码，而是"吃透"这次经历
- **完整解释前因后果** - 从需求到实现到结论
- **多用对比法** - 失败 vs 成功、误解 vs 正解
- **显式指出可迁移经验** - 知道"这次怎么修"，更要知道"以后怎么想"

## 示例

### 修复 bug 后
```
/teach-after "数据库连接池耗尽的问题"
```

### 功能开发完成后
```
/teach-after "用户认证功能"
```
